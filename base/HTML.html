    # meta viewport

    用于在移动端浏览器中更好地去控制布局，移动端 viewport 默认值为 980px, meta 元素可以用来改变移动端默认的 viewport 和缩放等设置， 以便在响应式的网页中正常显示。

    <!--视口宽度按照设备宽度去渲染， 初始缩放至为1，用户可以正常放大/缩小 -->
<meta name="viewport" content="width=device-width, initial-scale=1" />
<!-- 
  user-scalable 是否允许用户缩放， 
  minimum-scale最小缩放值， maximum-scale最大缩放值， 
  viewport-fit 用于适配iphoneX的刘海屏问题 
-->
<meta
  name="viewport"
  ccontent="width=device-width, 
  initial-scale=1, minimum-scale=1, maximum-scale=1, 
  user-scalable=no, viewport-fit=cover"
/>

    Document Object Model 文档对象模型
    最顶层的 document 节点是 document.documentElement,这是对应<html>标签的 DOM 节点。

    <body> = document.body

    # childNodes、firstChild、lastChild

    childNodes 看起来是一个数组，实际上不是，而是一个集合， 一个类数组的可迭代对象。 集合列出了所有子节点， 包括文本节点、元素节点、注释节点等。
    firstChild 和 lastChild 属性是访问第一个和最后一个子元素的快捷方式。

    elem.childNodes[0] === elem.firstChild;
elem.childNodes[elem.childNodes.length - 1] === elem.lastChild;
elem.hasChildNodes(); // 检查节点是否有子节点

    # table 的一些属性

    <table>元素支持 (除了上面给出的，之外) 以下属性:
    table.rows —— <tr> 元素的集合。
    table.caption/tHead/tFoot —— 引用元素 <caption>，<thead>，<tfoot>。
    table.tBodies —— <tbody> 元素的集合（根据标准还有很多元素，但是这里至少会有一个 —— 即使没有被写在 HTML 源文件中，浏览器也会将其放入 DOM 中）。
    <thead>，<tfoot>，<tbody> 元素提供了 rows 属性：
    tbody.rows —— 表格内部 <tr> 元素的集合。
    <tr>：
    tr.cells —— 在给定 <tr> 中的 <td> 和 <th> 单元格的集合。
    tr.sectionRowIndex —— 给定的 <tr> 在封闭的 <thead>/<tbody>/<tfoot> 中的位置（索引）。
    tr.rowIndex —— 在整个表格中 <tr> 的编号（包括表格的所有行）。
    <td> 和 <th>：
    td.cellIndex —— 在封闭的 <tr> 中单元格的编号。

    # DOM 属性

    当浏览器加载页面时，它会“读取”（或者称之为：“解析”）HTML 并从中生成 DOM 对象。对于元素节点，大多数标准的 HTML 特性（attributes）会自动变成 DOM 对象的属性（properties）。
    例如，如果标签是 <body id="page">，那么 DOM 对象就会有 body.id="page"。
    elem.hasAttribute(name) ---检查特性是否存在
    elem.getAttribute(name) --- 获取这个特性值
    elem.setAttribute(name, value) --- 设置这个特性值
    elem.removeAttribute(name) --- 移除这个特性值
    elem.attributes --- 读取所有特性： 属于内建 Attr 类的对象的集合，具有 name 和 value 属性。

    <input />

<script>
  let input = document.querySelector('input');

  // 特性 => 属性
  input.setAttribute('id', 'id');
  alert(input.id); // id（被更新了）

  // 属性 => 特性
  input.id = 'newId';
  alert(input.getAttribute('id')); // newId（被更新了）
</script>
<input />

<script>
  let input = document.querySelector('input');

  // 特性 => 属性
  input.setAttribute('value', 'text');
  alert(input.value); // text

  // 这个操作无效，属性 => 特性
  input.value = 'newValue';
  alert(input.getAttribute('value')); // text（没有被更新！）
</script>

<script>
  // 这段代码找到带有标记的元素，并显示需要的内容
  let user = {

  for (let div of document.querySelectorAll('[show-info]')) {

    # 元素大小

    # offsetParent, offsetLeft/Top

    offsetParent是最接近的具有定位属性的祖先
    offsetLeft是当前元素相对于 offsetParent 元素 X 坐标距离
    offsetTop是当前元素相对于 offsetParent 元素 Y 坐标距离

    有以下几种情况下，offsetParent的值为 null:

    1. 对于未显示的元素（display:none 或者不在文档中）；
    2. 对于<body>和<html>；
    3. 对于带有position: fixed的元素

    # offsetWidth, offsetHeight

    元素外部的宽度/高度，完整大小（包括边框）

    # clientTop/Left

    元素内侧与外侧的相对左边，包含 border 宽度+滚动条宽度（如果有的话），不包括 padding

    # clientWidth/Height

    元素边框内区域大小，包括 contentWidth+padding, 不包含滚动条

    # scrollWidth/Height

    就像 clientWidth/clientHeight, 但还包括滚动出（隐藏）部分

    # scrollTop/Left

    元素隐藏、滚动部分的 width/height, 可修改（可赋值的）

    # Window 大小和滚动

    获取窗口(window)的宽度和高度：
    document.documentElement.clientWidth/clientHeight --- 不含滚动条，可视区域
    window.innerWidth/innerHeight --- 包含滚动条

    # scrollTo,scrollBy,scrollInfoView

    必须在 DOM 完全构建好了之后才能通过 js 滚动页面
    scrollBy(x,y) --- 将页面滚动至相对当前位置的（x,y)位置， 例如 window.scrollBy(0, 10)表示页面向下滚动 10px;
    scrollTo(x,y) --- 将页面滚动至绝对坐标，使得可见部分的左上角具有相对于文档左上角的坐标 (pageX, pageY)。就像设置了 scrollLeft/scrollTop 一样。
    elem.scrollIntoView(top) --- top 为 true 时，使元素 elem 出现在窗口顶部，元素上边缘与窗口齐平；top 为 false 时，使元素 elem 出现在窗口底部，元素下边缘与窗口齐平。

    # 禁止滚动

    # 坐标

    1. 相对于窗口，clientX/Y, 随着页面滚动，元素相对于窗口的坐标会变动；
    2. 相对于文档，pageX/Y，元素在文档中的坐标保持不变

    pageX=clientX+文档的水平滚动出的部分的宽度
    pageY=clientY+文档的高度滚动出的部分的高度

    elem.getBoundingClientRect()返回元素最小矩形的窗口坐标
    返回的属性：
    x/y --- 矩形原点（左上角）相对于窗口的 X/Y 坐标
    width/height --- 矩形的 width/height
    top/bottom --- 顶部/底部矩形边缘的 Y 坐标 bottom=height+y top = y
    left/right --- 左侧/右侧矩形边缘的 X 坐标 right=width+x left = x
    document.elementFromPoint(x,y) --- 返回在窗口坐标（x,y)处的元素, 如果任何坐标为负或者超过了窗口的 width/height，那么该方法就会返回 null。

    let centerX = document.documentElement.clientWidth / 2;
let centerY = document.documentElement.clientHeight / 2;

let elem = document.elementFromPoint(centerX, centerY);

elem.style.background = 'red';

    // 获取元素的文档坐标
function getCoords(elem) {
  let box = elem.getBoundingClientRect();

  return {
    top: box.top + window.pageYOffset,
    right: box.right + window.pageXOffset,
    bottom: box.bottom + window.pageYOffset,
    left: box.left + window.pageXOffset,
  };
}

function createMessageUnder(elem, html) {
  let message = document.createElement('div');
  message.style.cssText = 'position:absolute; color: red';

  let coords = getCoords(elem);

  message.style.left = coords.left + 'px';
  message.style.top = coords.bottom + 'px';

  message.innerHTML = html;

  return message;
}

    # 语义化标签

    语义化： 合理正确地使用语义化标签来创建页面结构；
    标签：header nav main article section aside footer
    好处：

    1. 在没有 CSS 的情况下，也能很好地呈现页面结构；
    2. 代码结构清晰，便于阅读；
    3. 易于开发和维护, 方便其他设备解析（如屏幕阅读器）根据语义渲染网页；
    4. 有利于搜索引擎优化 seo,搜索引擎爬虫会根据不同的标签来赋予不同的权重
