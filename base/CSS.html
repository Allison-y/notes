    # CSS 选择器

    参考链接 (opens new window)

    1. 通配符选择器 *

    2. 类名选择器

    3. id 选择器

    4. 标签选择器

    5. 属性选择器 [attribute] eg: [data-title]选择 含有 data-title 属性 的元素
      [attribute=value] eg: [data="aa"] 选择 data 属性为 aa 的元素
      [attribute~=value] eg: [data~="aa"] 选择 data 属性中含有aa属性值的元素
      [attribute|=value] eg: [data|="aa"] 选择 data 属性值以 aa 值开头的元素，aa 必须是完整的单词
      [attribute^=value] eg: [data^=aa] 选择 data 属性值以 aa 值开头的元素，aa 可以是单词前缀
      [attribute$=value] eg: [data$=aa] 选择 data 属性值以 aa 值结尾的元素
      [attribute*=value] eg: [data*=aa] 选择 data 属性值含有 aa 字符的所有元素

    6. 伪元素选择器 伪元素主要用于指定某些元素的特定部分，以表示无法用 HTML 语义表达的实体。
      伪类主要用于区分元素的不同状态或行为。
      通常情况下，伪元素前面是 ::，包含两个英文冒号；而伪类前面只有一个英文冒号 :。
      ::before ::after
      ::first-line 用于选择某个块级元素的第一行。
      ::first-letter 用于选择某个块级元素的的第一个文字，并且文字所处的行之前没有其他内容（如图片和内联的表格）。
      ::selection 用于选择用户已选取的部分，即页面高亮的部分。
      placeholder 通常用作表单元素的占位文本，因此 ::placeholder 用来设置占位文本的样式。
      ::marker 用于选中列表中标记的元素。它作用在任何设置了 display: list-item 的元素或伪元素上。
      滚动条相关的伪元素:
      ::-webkit-scrollbar —— 整个滚动条。
      ::-webkit-scrollbar 用于选中整个滚动条。该伪类尽量不要单独设置，要设置该伪类就需要在伪类
      ::-webkit-scrollbar-thumb 中设置滚动条滑块的背景颜色，否则滚动条的滑块就会变透明。
      ::-webkit-scrollbar-button —— 滚动条上的按钮（上下箭头）。
      ::-webkit-scrollbar-thumb —— 滚动条上的滚动滑块。
      ::-webkit-scrollbar-track —— 滚动条轨道（下图浅蓝色部分）。
      ::-webkit-scrollbar-track-piece —— 滚动条没有滑块的轨道部分。
      ::-webkit-scrollbar-track-piece:start —— 滚动条没有滑块的轨道部分，滑块的前面（横向滚动条滑块左边、纵向滚动条滑块上面）。
      ::-webkit-scrollbar-track-piece:end —— 滚动条没有滑块的轨道部分，滑块的后面（横向滚动条滑块右边、纵向滚动条滑块下面）
      ::-webkit-scrollbar-corner —— 当同时有垂直滚动条和水平滚动条时交汇的部分。通常是元素的右下角。

    7. 伪类选择器 :root 选中文档的根元素。对于 HTML 来说，:root 表示 <html> 元素，除了优先级更高之外，与 <html> 选择器相同。在声明全局 CSS 变量时 :root 会很有用：
      :link 选中未访问过的链接标签，一般作用于带有 herf 属性的 <a> 标签和 <area> 标签。
      :visited 选中已访问过的链接元素 顺序:link — :visited — :hover — :active
      :visited 伪类和 :link 伪类是互斥的。
      :target 表示一个唯一的元素（目标元素），其 id 与当前 URL 片段匹配。当页面 URL 的 hash 为 #test 时，即页面的 location.hash === '#test' 时，id 为 test 的元素就会红色高亮。
      :not(selector) 用来匹配不符合一组选择器的元素（除了括号内中选择器匹配的元素之外的其他元素）。由于它的作用是防止特定的元素被选中，它也被称为反选伪类。
      :is(selector) 该函数将选择器列表作为参数，并选择该列表中任意一个选择器可以选择的元素。

      ul li,
ol li {
}

:is(ul, ol) li {

      :nth-child(n) 伪类选择父元素的第 n 个子元素。n 可以是任何正整数，也可以是表达式。

    8. 组合选择器 后代选择器 (selectorA selectorB)
      子元素选择器 (selectorA > selectorB)
      相邻兄弟选择器 (sA + sB) 选择上一个选择器为 sA 的 sB 元素, 要紧挨着

    # CSS 选择器优先级及权重

    important > 内联样式(1000) > id 选择器(100) > 类名选择器(10) = 伪类选择器 = 属性选择器 > 标签选择器(1) = 伪元素选择器 > 通配符选择器

    # 元素定位 position

    relative 相对定位
    absolute 绝对定位， absolute 定位使元素的位置与文档流无关，因此不占据空间。
    fixed 固定定位， Fixed 定 位使元素的位置与文档流无关，因此不占据空间。
    sticky 粘性定位，可以说是相对定位 relative 和固定定位 fixed 的结合；它主要用在对 scroll 事件的监听上；简单来说，在滑动过程中，某个元素距离其父元素的距离达到 sticky 粘性定位的要求时(比如 top：100px)；position:sticky 这时的效果相当于 fixed 定位，固定到适当位置。
    static 默认值。没有定位

    # 盒子模型 box-sizing

    content-box 标准盒子模型，css 设置的 height/width 不包含 padding 和 border
    border-box IE/怪异 盒子模型， css 设置的 height/width 包含 padding 和 border

    # BFC

    block formatting context container 块级格式化上下文，是 CSS 布局的一个概念，是一个独立的渲染区域，规定了内部 box 布局和排列方式，并且这个区域的子元素不会影响到外面的元素， 其中比较重要的布局规则有内部 box 垂直放置，计算 BFC 的高度时，浮动元素也参与计算。

    如何创建 BFC?

    1. 根元素，即 html 元素
    2. float 不为 none
    3. display 是 inline-block/table-cell/table-caption
    4. position 为 absolute/fixed
    5. overflow 不为 visible

    BFC 作用/使用场景?

    1. 清除浮动带来的影响, 确保父元素的高度包含子浮动元素， 避免元素被浮动元素覆盖，避免父元素高度崩塌
    2. 去除边距重叠现象，在同一个 BFC 容器内的相邻块元素的边距不会重叠，防止元素垂直边距塌陷问题（比如： 父元素的 marginTop 为 100， 子元素 marginTop 为 20， 如果没有创建 BFC, 呈现出来的就是父子元素边距重叠，全部都是 marginTop=100）

    # 清除浮动影响的方法

    1. 给父元素设置高度或者添加 overflow 属性；
    2. 在最后一个浮动元素后面再加一个元素，设置样式：(两边都有浮动)style=“clear：both”, (左浮动)clear:left, 右浮动：clear:right；
    3. 在下方被影响的元素上添加 clear:both;
    4. 在父容器的伪元素上做清除浮动 clear:both

    # 元素水平垂直居中

    水平居中
    对于行内元素：
    text-align: center

    对于确定宽度的块级元素：

    1. margin: 0 auto;
    2. 父相对定位，子绝对定位和 margin-left

    对于不确定宽度的块级元素：

    1. 父相对定位+子绝对定位和 transform;
    2. display:flex + justify-content: center;
    3. (table 标签/display：table)配合 margin 左右 auto 实现水平居中;
    4. display:inline-block + text-align:center

    垂直居中

    1. 父相对定位+子绝对定位和 transform;
    2. flex 布局 + align-items: center;
    3. 纯文字： line-height;
    4. 父相对定位+子绝对定位和 margin;
    5. 父 flex 布局+子：margin:auto;
    6. table 布局，子级设置 vertical-align 实现，需要注意的是：vertical-align: middle 使用的前提条件是内联元素以及 display 值为 table-cell 的元素

    # flex 布局

    align-content和align-items的区别
    参考链接 (opens new window)
    align-content 相对的是 flex 容器, 只有在 flex-wrap 设置为 wrap/wrap-reverse 时生效
    align-items 相对的是单个 flex 元素容器

    一些忽略的点
    justify-content
    space-around 在每行上均匀分配弹性元素。相邻元素间距离相同。每行第一个元素到行首的距离和每行最后一个元素到行尾的距离将会是相邻元素之间距离的一半。

    flex-item 的一些属性
    flex-grow 用来设置项目在 flex 容器中分配剩余空间的相对比例。默认为 0，表示不放大。
    flex-shrink flex 元素仅在默认宽度之和大于容器的时候才会发生收缩，其收缩的大小是依据 flex-shrink 的值。默认为 1，即如果空间不足会自动缩小。
    flex-basis指定了 flex 元素在主轴方向上的初始大小。如果不使用 box-sizing 改变盒模型的话，那么这个属性就决定了 flex 元素的内容盒(content-box)的尺寸。默认为 auto，即根据内容自适应。
    当一个元素同时被设置了 flex-basis (除值为 auto 外) 和 width (或者在 flex-direction: column 情况下设置了 height) , flex-basis 具有更高的优先级。
    flex是flex-grow、flex-shrink、flex-basis的简写形式，默认值为 0 1 auto。
    flex-flow 是 flex-direction 和 flex-wrap 属性的简写，默认值是 row nowrap。

    # rem 布局

    以 html font-size 为基准的倍数关系；需要给 html 设置一个基础的 font-size,一般选屏幕宽度的 1/10
    其实布局分布就是按照 fontSize 的比例实现， 也就是按照屏幕的大小比例去实现
    优点
    可以快速适用移动端布局，字体，图片大小
    缺点

    1. 目前 ie 不支持, 对 pc 页面来讲使用次数不多；
    2. 数据量大，所有的图片、盒子都需要我们去给一个准确的值，才能保证不同机型的适配；
    3. 在响应式布局中，必须通过 js 来动态控制根元素 font-size 的大小。也就是说 css 样式和 js 代码有一定的耦合性。且必须将改变 font-size 的代码放在 css 样式之前。

    # grid 布局

    参考链接 (opens new window)
    网格布局
    display 将元素定义为网格容器，并为其内容建立新的网格格式上下文。
    属性值：
    grid 生成块级网格
    inline-grid 生成内联级网格
    grid-template-columns/grid-template-rows
    属性值：
    trick-size 可以是长度、百分比。也可以是网格中可用空间的一部分，使用 fr 单位。
    line-name 可自定义名称

    /* grid-template-columns有5个值，表示生成了5列，每列的宽度依次为40px、50px、auto 50px 40px。
 * grid-template-rows有3个值。表示生成了3行，每行的高度依次为25%、100px、auto。
*/
.container {
  grid-template-columns: 40px 50px auto 50px 40px;
  grid-template-rows: 25% 100px auto;
}
/**
 * 给网格分隔线进行命名，语法是使用[]包裹我们自定义的命名。
 * 一行可以有多个名称，如：这里的第二行可以有两个名称row1-end和row2-start
*/
.container {
  grid-template-columns: [first] 40px [line2] 50px [line3] auto [col4-start] 50px [five] 40px [end];
  grid-template-rows: [row1-start] 25% [row1-end] 100px [third-line] auto [last-line];
}
/**
 * 若定义中存在重复的部分，可以用repeat()符号来简化
*/
.container {
  grid-template-columns: repeat(3, 20px[col-start]);
}
/*相当于*/
.container {
  grid-template-columns: repeat(20px [col-start] 20px [col-start] 20px [col-start]);
}

    justify-items
    沿行（水平）轴对齐网格项，此值适用于容器内的所有网格项。
    属性值：
    start 对齐项目，使其与单元格的起始边缘齐平
    end 使项目与其单元格的末端边缘齐平
    center 将项目排列在单元格的中央
    stretch 填充单元格的整个宽度（默认值）
    也可以通过 justify-self 属性在单个网格项上设置这种行为。

    align-items
    沿列（垂直）轴对齐网格项，此值适用于容器内的所有网格项。
    属性值：
    start 对齐项目，使其与单元格的起始边缘齐平
    end 使项目与其单元格的末端边缘齐平
    center 将项目排列在单元格的中央
    stretch 填充单元格的整个宽度（默认值）
    baseline 沿文本基线对齐项目。基线有修改器--第一基线和最后基线，在多行文本的情况下，将使用第一行或最后一行的基线。
    也可以通过 align-self 属性在单个网格项上设置这种行为。

    place-items
    在一个声明中同时设置了 align-items 和 justify-items 属性。

    .container {
  place-items: <align-items> / <justify-items>;
}
/* 快速水平居中 */
.container {
  display: grid;
  place-items: center;
}

    justify-content
    有时，网格的总大小可能小于网格容器的大小。如果所有网格项的大小都使用非灵活单位（如 px），就会出现这种情况。在这种情况下，你可以设置网格在网格容器中的对齐方式。
    该属性使网格沿水平（行）轴对齐。
    属性值：
    start 对齐网格，使其与网格容器的起始边缘齐平
    end 使网格与网格容器的末端边缘齐平
    center 将网格对齐到网格容器的中央
    stretch 调整网格项的大小，使网格填满网格容器的整个宽度
    space-around 在每个网格项之间留出均等的空间，在两端留出一半大小的空间
    space-between 在每个网格项之间留出均等的空间，远端不留空间
    space-evenly 在每个网格项（包括远端）之间留出均等的空间

    align-content
    该属性可使网格沿垂直（列）轴对齐。
    属性值：
    start 对齐网格，使其与网格容器的起始边缘齐平
    end 使网格与网格容器的末端边缘齐平
    center 将网格对齐到网格容器的中央
    stretch 调整网格项的大小，使网格填满网格容器的整个宽度
    space-around 在每个网格项之间留出均等的空间，在两端留出一半大小的空间
    space-between 在每个网格项之间留出均等的空间，远端不留空间
    space-evenly 在每个网格项（包括远端）之间留出均等的空间

    place-content
    place-content 在一个声明中同时设置了 align-content 和 justify-content 属性。
    <align-content> / <justify-content> – 第一个值设置 align-content，第二个值设置 justify-content。如果省略第二个值，则会将第一个值分配给这两个属性。

    grid-auto-flow
    如果有网格项没有明确放置在网格上，自动放置算法就会启动，自动放置网格项。此属性控制自动放置算法的工作方式。
    属性值：
    row 告诉自动放置算法依次填充每一行，必要时添加新行（默认值）
    column 告诉自动置放算法依次填充每一列，并根据需要添加新列
    dense 告诉自动置放算法，如果后面出现较小的项目，则尝试在网格中较早的位置填孔

    网格项上的属性

    grid-column-start/grid-column-end/grid-row-start/grid-row-end

    grid-column-start/grid-row-start 是项目开始所在的网格线
    grid-column-end/grid-row-end 是项目结束所在的网格线。

    .item {
  grid-column-start: <number> | <name> | span <number> | span <name> | auto;
  grid-column-end: <number> | <name> | span <number> | span <name> | auto;
  grid-row-start: <number> | <name> | span <number> | span <name> | auto;
  grid-row-end: <number> | <name> | span <number> | span <name> | auto;
}
.item-a {
  grid-column-start: 1;
  grid-column-end: span col4-start; // 项目将横跨网格，直到到达下一条名称为col4-start的网格线为止
  grid-row-start: 2;
  grid-row-end: span 2; // 项目将跨越2个网格线
}

    特殊的单位和方法

    fr

    fr 是单词 fraction 的缩写，表示分数。

    .container {
  grid-template-columns: 1fr 1fr 1fr; // 表示三等分
}
.container {
  grid-template-columns: 50px 1fr 1fr 1fr; // 表示第一列固定宽，剩余部分等分
}
/* 当设置fr尺寸的时候，auto的尺寸表现为“包裹”，为内容宽度。如果没有设置fr尺寸的网格，则表现为拉伸。 */
.container {
  grid-template-columns: auto 1fr 1fr 1fr;
